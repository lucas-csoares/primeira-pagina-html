    2. O resultado é a estrutura básica HTML conforme Figura 1.0
  4. Navegador (preferencialmente google chrome)

Inspetor de elementos

Com a ferramenta DevTools (ou inspetor de elementos) é possível visualizar o código-fonte de qualquer página web. Além disso, é possível fazer alterações temporárias e ver o resultado em tempo real, deixando o código-fonte original intacto.

Isso é muito útil para diagnosticar eventuais problemas em páginas Web ou testar uma alteração. Também pode ser útil se você se depara com um recurso de uma página Web, acha interessante e quer ver como ele foi implementado.

Como inspecionar elemento no Chrome?

Existe 3 maneiras para inspecionar elemento no Chrome.

  • Clique com o botão direito em um item da página e escolha Inspecionar;
  • Clique no menu de três pontos no canto superior direito da janela e escolha Mais ferramentas > Ferramenta de desenvolvedor;
  • Pressione Ctrl + Shift + C no teclado (Cmd + Option + C no Mac).

Quais são os recursos da ferramenta?

  • Encontrar classes CSS em um site
  • Solucionar problemas de um site
  • Editar o texto para visualizar as alterações
  • Visualizar as alterações na imagem
  • Testar o site usando emulação de dispositivo

Estrutura básica do HTML

Aqui vamos explorar o entendimento básico por trás da estrutura HTML, como uma página HTML típica é estruturada e explicaremos outras importantes características básicas da linguagem

O que é HTML?

HTML é um documento de texto simples estruturado com elementos. Tais elementos são acompanhados de abertura e fechamento de tags. Você pode visualizar a anatomia de um elemento HTML na Figura 2.0

Um arquivo HTML é salvo com extensão .html e pode ser renderizado por qualquer browser

Outro aspecto importante da linguagem HTML é que ela é case insensitive, ou seja, não leva em consideração a diferença entre letras maiúsculas e minúsculas.

Documento HTML

Como os elementos individuais são combinados de forma a criar uma página HTML inteira?

Analisando a Figura 1.0 temos o seguinte:

  1. <!DOCTYPE html>: Antigamento o doctype funcionava como links funcionava como links para várias regras as quais o HTML tinha que seguir. No entanto, hoje em dia ele não se aplica e precisa ser incluído para que tudo funcione da forma correta.
  2. <html></html>: envolve o conteúdo da página HTML inteira e é conhecido como elemento raiz da página.
  3. <head></head>: O elemento <head> é um container para todo o conteúdo da página HTML que não é visível pelo usuário. Isso inclui palavras-chave e a descrição que você quer que apareça nos resultados de busca.
  4. <meta charset="utf-8">: define o tipo de codificação dos caracteres que o documento deve usar, neste cas o UTF-8, que inclui a maior parte dos caracteres das línguas humanas. Tal elemento ajuda a evitar vários problemas no site.
  5. <title></title>: Define o título da página que vai aparecer na guia do navegador na qual a página está carrregada.
  6. <body></body>: contém todo o conteúdo que você quer exibir ao usuário quando eles visitarem a página.

Falando sobre tags

Na Firua 2.0 vimos a anatomia de um elemento HTML. Note que uma das propriedades do elemento é justamente a tag. Existe a tag de abertura e de fechamento.

Atributos de tags

É justamente na tag de abertura que existe os atributos.

Na Figura 3.0 abaixo é mostrado como o atributo se apresenta:

Os atributos contém informações sobre os elementos que você não deseja que apareça no conteúdo. Na Figura 3.0 temos o atributo class que permite que você dê ao elemento um nome de identificação, como se fosse um ID para ser usado no CSS como forma de estilizá-lo.

Regras do atributo

  1. Um espaço entre ele e o nome do elemento (ou o atributo anterior)
  2. O nome do atributo, seguido por um sinal de igual
  3. Um valor para o atributo, com as de abertura e fechamento em volta dele

Deve-se usar <font> estilização CSS para configurar estilo de fonte para o texto

Aninhando elementos

Elementos podem ser inseridos dentro de outros. Isto se chama aninhamento.

Elementos em bloco x Elementos inline

  • Elementos em bloco formam um bloco visível na página — eles aparecerão em uma nova linha logo após qualquer elemento que venha antes dele, e qualquer conteúdo depois de um elemento em bloco também aparecerá em uma nova linha
  • Elementos inline (na linha) são aqueles que estão contidos dentro de elementos em bloco envolvem apenas pequenas partes do conteúdo do documento e não parágrafos inteiros ou agrupamentos de conteúdo. Um elemento inline não fará com que uma nova linha apareça no documento: os elementos inline geralmente aparecem dentro de um parágrafo de texto, por exemplo: um elemento <a>, ou elementos de ênfase como <em> ou <strong>

Boas práticas de hierarquia estrutural

  • Usar apenas <h1> por página. Esse é o título superior e todos ficam abaixo dele na hierarquia.
  • Use títulos na ordem correta na hierarquia (não usar um h3 seguido de h2)
  • Pra cada página HTML deve-se usar no máximo 3 dos 6 níveis de cabeçalho disponíveis (A menos que seja muito necessário)
